미들웨어와 라우팅
- Express.js는 미들웨어와 라우팅을 통해 웹 애플리케이션의 동작을 제어
- 미들웨어는 요청(Request)과 응답(Response) 사이에서 동작하는 함수로, Express.js에서 강력한 기능을 제공하며 웹 애플리케이션의 동작을 제어하는데 사용
- 미들웨어는 웹 요청을 처리하고, 다음 미들웨어에게 요청을 전달하거나 응답을 생성하여 클라이언트에 반환
- 라우팅은 클라이언트의 요청에 따라 적절한 핸들러 함수를 호출하는 과정을 의미
- Express.js에서는 app.get(), app.post(), app.put() 등의 메서드를 사용하여 라우팅을 정의
- 각각의 라우팅 메서드는 요청된 URL 경로와 해당 경로에 대한 핸들러 함수를 매핑
- 클라이언트가 특정 URL 경로에 요청을 보내면 Express.js는 해당 경로에 등록된 핸들러 함수를 실행하여 요청을 처리하고 응답을 반환

미들웨어와 라우팅 사용법
- 미들웨어를 사용하기 위해서는 app.use() 메서드를 사용하여 미들웨어 함수를 등록
- 미들웨어 함수는 req (요청 객체), res (응답 객체), next (다음 미들웨어를 호출하는 함수)를 인자로 받음
- 미들웨어 함수는 요청을 처리하거나 다음 미들웨어로 요청을 전달할 수 있습니다.

app.use((req, res, next) => {
  // 미들웨어 로직 수행
  // 다음 미들웨어 호출을 위해 next() 호출
});

- 라우팅을 정의하기 위해서는 app.get(), app.post(), app.put() 등의 메서드를 사용함 / 각각의 메서드는 경로와 해당 경로에 대한 핸들러 함수를 매핑
- 핸들러 함수는 클라이언트의 요청을 처리하고 응답을 생성

app.get('/users', (req, res) => {
  // 요청된 URL 경로에 대한 로직 수행
  // 응답 생성
});
