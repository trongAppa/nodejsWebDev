JSON-RPC의 구성 요소
요청 메시지와 응답 메시지로 구성되며, 각각은 JSON 객체로 표현
요청 메시지는 호출할 프로시저의 이름과 매개변수를 포함하고, 응답 메시지는 프로시저의 실행 결과를 포함
클라이언트와 서버 간의 통신을 위해 HTTP, TCP/IP 등 다양한 프로토콜을 사용할 수 있음

JSON-RPC의 구성 요소
1. JSON-RPC 버전 (jsonrpc)
- JSON-RPC 프로토콜의 버전 / 일반적으로 "2.0"을 사용

2. 메서드 (method)
- 원격 프로시저의 이름 / 클라이언트는 이를 통해 원격으로 호출하고자 하는 프로시저를 명시

3. 매개변수 (params)
- 원격 프로시저에 전달하는 매개변수 / 선택적인 필드이며, 프로시저에 전달할 데이터를 포함 / 매개변수는 배열 또는 객체 형태로 전달

4. 요청 식별자 (id)
- 요청을 고유하게 식별하기 위한 식별자 / 클라이언트가 요청을 보낼 때 생성하고, 서버는 이 식별자를 응답에서 그대로 반환 / 클라이언트는 요청과 응답을 연결하여 구분

5. 성공 응답 (result)
- 원격 프로시저의 실행 결과 / 성공적으로 실행된 경우, 해당 결과가 이 필드에 포함

6. 오류 응답 (error)
- 원격 프로시저 실행 중 발생한 오류 정보 / 오류가 발생하지 않은 경우, 이 필드는 생략 / 오류 응답은 code, message, data 등의 하위 필드를 포함
 - code : 오류 코드 / 일반적으로 정수 값으로 표현되며, 특정 오류를 식별하는 데 사용
 - message : 오류에 대한 간단한 설명을 제공
 - data : 추가적인 오류 정보를 포함하는 필드로, 선택적으로 사용
 
구성 요소들을 활용하여 클라이언트와 서버 간의 요청과 응답을 교환
클라이언트는 요청 메시지를 작성하고, 서버는 이를 받아 해당하는 원격 프로시저를 실행
서버는 응답 메시지를 생성하여 클라이언트로 반환하고, 클라이언트는 이를 처리하여 결과를 활용
원격 프로시저 호출을 통한 서버와의 통신을 간편하게 구현
