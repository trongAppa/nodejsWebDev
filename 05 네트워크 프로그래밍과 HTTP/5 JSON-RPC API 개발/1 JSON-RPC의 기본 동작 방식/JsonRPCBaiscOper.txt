JSON-RPC의 기본 동작 방식
클라이언트가 JSON 형식의 요청 메시지를 생성하고, 이를 서버에 전송하여 프로시저를 호출
서버는 해당 요청을 처리하고, 결과를 JSON 형식의 응답 메시지로 반환
클라이언트와 서버 간의 프로시저 호출을 간편하게 수행

원격 프로시저 호출(Remote Procedure Call) 프로토콜로, 클라이언트와 서버 간의 통신을 위한 방식

1. 요청 작성
- 클라이언트는 JSON-RPC 요청 메시지를 작성
- 요청 메시지는 JSON 객체로 구성되며, 반드시 jsonrpc, method, params, id 등의 필드를 포함
 - jsonrpc : 사용하는 JSON-RPC 버전 / 일반적으로 "2.0"을 사용
 - method : 호출하려는 원격 프로시저의 이름
 - params : 원격 프로시저에 전달할 매개변수를 포함하는 필드
 - id : 요청을 식별하기 위한 고유한 식별자

2. 요청 전송
- 클라이언트는 요청 메시지를 서버로 전송
- 일반적으로 HTTP POST 요청을 사용하며, 요청 본문에 요청 메시지를 포함

3. 서버 처리
- 서버는 요청을 받아 해당하는 원격 프로시저를 실행
- 요청 메시지의 method 필드에 해당하는 프로시저를 호출하고, params 필드의 매개변수를 전달

4. 응답 생성
- 서버는 원격 프로시저의 실행 결과에 따라 응답 메시지를 생성
- 응답 메시지도 JSON 객체로 구성되며, jsonrpc, result, error, id 등의 필드를 포함
 - jsonrpc : 사용하는 JSON-RPC 버전 / 일반적으로 "2.0"을 사용
 - result : 원격 프로시저의 실행 결과 / 성공적인 경우에는 해당 결과가 포함
 - error : 원격 프로시저 실행 중 발생한 오류 정보를 포함 / 오류가 발생하지 않은 경우에는 이 필드가 생략
 - id : 요청 메시지의 id와 동일한 값을 가지며, 요청을 식별하기 위해 사용

5. 응답 전송
- 서버는 응답 메시지를 클라이언트로 전송
- 일반적으로 HTTP 응답으로 반환되며, 응답 본문에 응답 메시지를 포함

6. 응답 처리
- 클라이언트는 서버로부터 받은 응답을 처리
- 응답 메시지의 result 필드에는 원격 프로시저의 실행 결과가 포함
- error 필드가 있는 경우, 오류 정보가 포함
- 클라이언트는 이를 적절히 처리하고 결과를 활용

클라이언트와 서버 간의 단순한 요청과 응답을 처리하는 프로토콜로, HTTP를 기반으로 동작하며, 데이터 포맷으로는 JSON을 사용
이를 이용하여 원격 프로시저 호출을 통해 서버와 통신하는 기능을 구현
