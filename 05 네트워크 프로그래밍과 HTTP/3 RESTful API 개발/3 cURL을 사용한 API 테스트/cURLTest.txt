cURL을 사용한 API 테스트
cURL은 명령 줄 인터페이스를 통해 HTTP 요청을 보내고 응답을 받을 수 있는 강력한 도구
cURL을 사용하여 API를 테스트할 때 
-i 옵션을 사용하면 응답의 헤더 정보를 확인
-X 옵션을 사용하여 요청 메소드를 지정
-H 옵션을 사용하여 요청 헤더를 설정
-d 옵션을 사용하여 요청 데이터를 전송

1. GET 요청 보내기
curl -i http://localhost:3000/api/users

-i 옵션을 사용하여 응답 헤더를 포함한 전체 응답을 출력 http://localhost:3000/api/users는 테스트하려는 API 엔드포인트의 주소
실행하면 해당 API 엔드포인트로 GET 요청이 전송되고 응답이 출력

2. POST 요청 보내기:
curl -i -X POST -H "Content-Type: application/json" -d '{"name": "John", "email": "john@example.com"}' http://localhost:3000/api/users

-X POST 옵션을 사용하여 POST 요청을 전송 
-H 옵션을 사용하여 요청 헤더를 설정
-d 옵션을 사용하여 JSON 형식의 요청 데이터를 전송 http://localhost:3000/api/users는 테스트하려는 API 엔드포인트의 주소
실행하면 해당 API 엔드포인트로 POST 요청이 전송되고 응답이 출력

3. PUT 요청 보내기
curl -i -X PUT -H "Content-Type: application/json" -d '{"name": "John Doe", "email": "johndoe@example.com"}' http://localhost:3000/api/users/1

-X PUT 옵션을 사용하여 PUT 요청을 전송
-H 옵션을 사용하여 요청 헤더를 설정
-d 옵션을 사용하여 JSON 형식의 요청 데이터를 전송 http://localhost:3000/api/users/1는 테스트하려는 API 엔드포인트의 주소
실행하면 해당 API 엔드포인트로 PUT 요청이 전송되고 응답이 출력

4. DELETE 요청 보내기
curl -i -X DELETE http://localhost:3000/api/users/1

-X DELETE 옵션을 사용하여 DELETE 요청을 전송

http://localhost:3000/api/users/1는 테스트하려는 API 엔드포인트의 주소
실행하면 해당 API 엔드포인트로 DELETE 요청이 전송되고 응답이 출력

5. JSON 형식의 데이터를 요청 바디에 추가하여 요청 보내기
curl -i -X POST -H "Content-Type: application/json" -d '{"jsonrpc": "2.0", "method": "subtract", "params": {"minuend": 42, "subtrahend": 23}, "id": 1}' http://localhost:3000/jsonrpc

-X POST 옵션을 사용하여 POST 요청을 전송
-H 옵션을 사용하여 요청 헤더의 Content-Type을 application/json로 설정
-d 옵션을 사용하여 JSON 형식의 요청 데이터를 전송 http://localhost:3000/jsonrpc은 JSON-RPC 엔드포인트의 주소

cURL을 사용하여 API를 테스트하고 디버깅하는 방법을 설명
API의 엔드포인트와 요청 방식에 따라 적절한 cURL 명령을 사용하여 테스트 가능
필요에 따라 다양한 옵션을 사용하고, 요청 헤더와 데이터를 설정하여 테스트 가능
