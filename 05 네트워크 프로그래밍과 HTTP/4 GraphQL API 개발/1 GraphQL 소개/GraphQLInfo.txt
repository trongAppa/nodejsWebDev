GraphQL
페이스북에서 개발된 쿼리 언어
웹 클라이언트가 서버로부터 원하는 데이터를 효율적으로 요청하고 받을 수 있게 도와줌

GraphQL의 기본 개념
스키마 (Schema)
- GraphQL은 스키마를 사용하여 서버의 데이터 구조를 정의
- 사용 가능한 타입과 타입 간의 관계를 정의하는데 사용
- GraphQL API의 계약서 역할을 하며, 클라이언트가 어떤 데이터를 요청할 수 있는지를 명시

쿼리 (Query)
- 클라이언트는 GraphQL API에 대해 쿼리를 작성하여 원하는 데이터를 요청
- 원하는 데이터의 구조를 지정하고 필요한 필드를 선택하는 방식으로 작성
- 클라이언트는 정확히 필요한 데이터만 요청할 수 있으므로, 불필요한 데이터의 오버헤드를 줄일 수 있음

리졸버 (Resolver)
- GraphQL API에서 쿼리에 대한 응답을 생성하는 로직을 담당
- 각 필드에 대한 리졸버 함수를 작성하여 해당 필드의 데이터를 가져오거나 가공
- 리졸버 함수는 서버에서 데이터를 가져오는 역할을 수행하며, 필요에 따라 외부 데이터 소스와 통신

GraphQL의 동작 방식
클라이언트는 GraphQL API에 대한 쿼리를 작성하고 전송
서버는 받은 쿼리를 해석하고 해당 쿼리에 대한 응답을 생성하기 위해 리졸버 함수를 실행
리졸버 함수는 필요한 데이터를 가져와 응답을 구성
서버는 클라이언트에게 구성된 응답을 반환
클라이언트는 받은 응답을 처리하고 필요한 데이터를 사용

GraphQL의 장점
Over-fetching과 Under-fetching 문제를 해결
클라이언트는 정확히 필요한 데이터만 요청할 수 있으므로, 필요한 데이터의 양을 최소화
여러 데이터 소스로부터 데이터를 가져올 수 있음
리졸버 함수를 사용하여 데이터를 가져오는 로직을 작성할 수 있으며, 필요에 따라 데이터베이스, 웹 서비스, 파일 시스템 등 다양한 소스로부터 데이터를 획득
단일 요청으로 여러 데이터를 한 번에 가져올 수 있습니다. 쿼리에서 필요한 여러 필드를 동시에 요청할 수 있으므로, 여러 요청을 병합하여 효율적인 데이터 전송

GraphQL은 유연하고 효율적인 데이터 요청을 가능하게 해주는 강력한 도구
