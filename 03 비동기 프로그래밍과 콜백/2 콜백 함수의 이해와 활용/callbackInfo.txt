콜백 함수의 이해와 활용
- 비동기 프로그래밍에서 중요한 개념으로, 비동기 작업이 완료되었을 때 호출되는 함수
- 사용하여 비동기 작업의 결과를 처리하거나 다음 작업을 수행할 수 있음
- 일반적으로 비동기 작업을 수행하는 함수의 인자로 전달 / 비동기 작업이 완료되면 해당 함수에서 콜백 함수를 호출하고, 필요한 데이터를 인자로 전달 / 비동기 작업의 결과를 콜백 함수에서 처리할 수 있음
- 익명 함수로 정의될 수도 있고, 선언된 함수를 직접 전달할 수도 있음 / 익명 함수로 정의할 경우에는 작업이 완료되었을 때 한 번만 호출되는 일회성 콜백 함수 / 선언된 함수를 전달할 경우에는 재사용 가능한 콜백 함수로 활용
- 활용은 다양한 형태로 이루어짐 / 예를 들어, 데이터베이스 조회 작업이 완료되었을 때 콜백 함수를 호출하여 결과를 처리하거나, 파일을 읽거나 쓰는 작업이 완료되었을 때 콜백 함수를 호출하여 다음 작업을 수행
- 사용할 때에는 몇 가지 주의할 점
 1. 콜백 함수의 실행 시점과 결과 처리를 보장하기 위해 콜백 함수 안에서 예외 처리 필요 
 2. 콜백 함수의 중첩 사용은 코드의 가독성을 저해할 수 있으므로 콜백 지옥을 피하기 위해 비동기 제어 흐름을 관리하는 방법을 고려
- Node.js에서 비동기 프로그래밍을 구현하는 데 핵심적인 역할 / 이해하고 활용하는 것은 Node.js 애플리케이션 개발에서 필수적인 요소 / 콜백 함수의 개념과 활용 방법을 숙지하여 비동기 작업을 효율적으로 처리할 수 있도록 해야 함

예제 코드
// 비동기 작업을 수행하는 함수
function asyncTask(callback) {
  setTimeout(() => {
    const result = '비동기 작업 완료';
    callback(result);
  }, 1000);
}

// 비동기 작업의 결과를 처리하는 콜백 함수
function handleResult(result) {
  console.log(result);
}

// 비동기 작업 수행
asyncTask(handleResult);

asyncTask 함수 : 비동기 작업을 수행하는 함수 / 1초 후에 '비동기 작업 완료'라는 결과를 생성하고, 인자로 전달된 콜백 함수인 callback을 호출하여 결과를 전달
handleResult 함수 : 비동기 작업의 결과를 처리하는 콜백 함수 / 이 함수는 결과를 인자로 받아와서 출력

asyncTask 함수를 호출하면서 handleResult 함수를 콜백으로 전달 -> 이렇게 하면 비동기 작업이 완료되면 handleResult 함수가 호출되어 결과가 출력
