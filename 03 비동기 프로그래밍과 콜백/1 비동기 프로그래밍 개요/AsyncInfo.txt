비동기 프로그래밍 개요
- 대기 시간이 필요한 작업을 효율적으로 처리하기 위한 프로그래밍 방식
- 동기적인 프로그래밍 방식은 작업을 순차적으로 실행하며, 한 작업이 끝나야 다음 작업을 수행할 수 있음 하지만 이러한 방식은 대기 시간이 필요한 작업에서는 비효율적
- 작업을 순차적으로 기다리지 않고, 다른 작업을 동시에 수행하여 대기 시간을 최소화하는 방식 / 작업의 처리 속도를 향상
- Node.js는 비동기 프로그래밍을 강력하게 지원하는 플랫폼 / Node.js는 단일 스레드로 동작하면서도 비동기 I/O 처리를 효율적으로 수행할 수 있음 / 가능하게 하는 주요 요소는 이벤트 루프와 콜백 함수입니다.
- 이벤트 루프는 Node.js의 핵심 개념 중 하나로, 비동기 작업의 완료를 감지하고 처리하는 역할 / 작업을 대기열에 추가하고, 이벤트가 발생하면 해당 작업을 실행
- 콜백 함수는 비동기 작업이 완료되었을 때 호출되는 함수 / 비동기 작업을 수행하는 함수에 콜백 함수를 전달하여 작업이 완료되면 콜백 함수가 실행 / 비동기 작업의 결과를 처리하거나 다음 작업을 수행
- 네트워크 요청, 파일 시스템 접근, 데이터베이스 쿼리 등의 작업에서 특히 유용 / 대기 시간이 필요하고, 동기적인 방식으로 처리하면 전체 애플리케이션이 블로킹되는 문제가 발생 / 비동기 프로그래밍을 통해 이러한 문제를 해결하고, 애플리케이션의 성능과 확장성을 개선
- Node.js에서의 비동기 처리 방식을 이해하면, 효율적인 작업 처리와 애플리케이션의 성능 개선을 위해 적절한 비동기 패턴과 도구를 활용

예제 코드
// 비동기적으로 숫자를 더하는 함수
function addNumbersAsync(a, b, callback) {
  setTimeout(() => {
    const result = a + b;
    callback(result);
  }, 1000);
}

// 비동기 작업의 결과를 처리하는 콜백 함수
function handleResult(result) {
  console.log('덧셈 결과:', result);
}

// 비동기 작업 수행
console.log('비동기 작업 시작');
addNumbersAsync(5, 3, handleResult);
console.log('비동기 작업 종료');

addNumbersAsync 함수는 비동기적으로 두 숫자를 더하는 함수
setTimeout을 사용하여 1초 후에 덧셈 결과를 생성하고, 결과를 인자로 전달된 콜백 함수 callback을 호출하여 전달
handleResult 함수는 비동기 작업의 결과를 처리하는 콜백 함수 / 이 함수는 결과를 인자로 받아와서 '덧셈 결과:'와 함께 출력합니다.

비동기 작업을 수행하는 부분에서 먼저 '비동기 작업 시작'이 출력 -> 그 후에 addNumbersAsync 함수가 호출되어 비동기 작업이 실행 -> '비동기 작업 종료'가 출력 -> 1초 후에 비동기 작업이 완료되고 handleResult 함수가 호출되어 결과가 출력
