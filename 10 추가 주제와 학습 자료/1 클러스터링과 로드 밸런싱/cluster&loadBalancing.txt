클러스터링과 로드 밸런싱
대규모 웹 서비스나 애플리케이션에서 발생하는 문제를 해결하기 위해서
이러한 시스템에서는 많은 트래픽이 발생하고, 단일 서버로는 처리하기 어려운 부하가 발생 / 클러스터링과 로드 밸런싱을 사용하여 이러한 문제들을 해결

클러스터링은 여러분들과 같은 뛰어난 분들이 한 팀을 이뤄 작업을 분산시키는 것과 비슷한 개념
트래픽이 많은 경우에도 여러분들의 리소스를 효과적으로 활용할 수 있고, 성능 저하를 방지
한 분이 문제가 생겼을 때 다른 분들이 그 역할을 대신 수행하여 서비스의 가용성을 높임

로드 밸런싱은 트래픽을 여러분들 중에서 고르게 분산시켜 병렬 처리를 가능
각 분들의 부하를 균등하게 분산시킬 수 있고, 응답 시간을 최소화하여 사용자에게 빠른 서비스를 제공
한 분의 문제로 인한 서비스 중단을 최소화하여 여러분들의 가용성을 향상

대규모 시스템에서 확장성, 가용성, 성능 향상을 위해 중요한 역할
안정적이고 빠른 서비스를 제공할 수 있으며, 여러분들의 사용자 경험을 향상

1) 클러스터링의 개념과 이점
- 여러 서버 인스턴스를 그룹화하여 단일 시스템처럼 동작하도록 만드는 것
- 애플리케이션의 가용성과 확장성을 향상
- 클러스터링의 이점과 다양한 클러스터링 아키텍처를 소개

2) 로드 밸런싱의 개념과 종류
- 트래픽을 여러 서버 인스턴스로 분산시켜 부하를 분담하는 것
- 로드 밸런서의 종류
 1. 라운드 로빈(Round Robin) 방식 : 트래픽을 순서대로 각 서버에 분배
 2. 가중치 기반(Weighted) 방식 : 서버에 가중치를 부여하여 트래픽을 분배
 3. 세션(Session) 기반 방식 : 사용자의 세션 정보를 기준으로 트래픽을 분배
- 로드 밸런서가 트래픽을 어떻게 분배하는지를 나타내며, 각 방식은 서버의 성능, 가용성, 사용자 경험 등을 고려하여 선택

3) Node.js 클러스터 모듈
- Node.js에는 클러스터 모듈이 내장되어 있어 클러스터를 쉽게 구현
- 클러스터 모듈을 사용하여 여러 워커 프로세스를 생성하고 관리하는 방법을 학습

4) Reverse Proxy와 로드 밸런서
- Reverse Proxy 서버와 로드 밸런서를 사용하여 트래픽을 분산시키는 방법
- Nginx, HAProxy와 같은 도구를 사용하여 로드 밸런싱과 리버스 프록시를 설정하는 방법

5) 클라우드 환경에서의 클러스터링과 로드 밸런싱
- 클라우드 제공 업체(AWS, Azure, GCP)에서 제공하는 클러스터링 및 로드 밸런싱 서비스를 활용하는 방법을 학습
- Elastic Load Balancer, Azure Load Balancer, GCP Load Balancer 등의 서비스를 사용하여 애플리케이션을 분산시키는 방법

6) 클러스터링과 로드 밸런싱의 모니터링
- 클러스터링과 로드 밸런싱을 운영하면서 모니터링을 수행하는 방법을 학습
- 서버 성능, 트래픽 분배, 인스턴스 상태 등을 모니터링하여 시스템의 안정성과 성능을 유지하는 방법
