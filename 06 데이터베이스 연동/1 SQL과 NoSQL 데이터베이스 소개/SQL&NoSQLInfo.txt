SQL과 NoSQL 데이터베이스 소개

SQL 데이터베이스 
- SQL(Structured Query Language)
- 관계형 데이터베이스
- 데이터를 테이블 형태로 구조화하고, 테이블 간의 관계를 설정하여 데이터를 저장하고 관리
- 데이터의 구조는 미리 정의된 스키마에 따라 이루어지며, 각 테이블은 열(Column)과 행(Row)으로 이루어진 데이터를 저장
- ACID(Atomicity, Consistency, Isolation, Durability) 특성을 가지며, 데이터의 일관성과 안정성을 보장
- 대표적으로 MySQL, PostgreSQL, Oracle 등이 있습니다.

NoSQL 데이터베이스
- NoSQL(Not Only SQL)
- 비관계형 데이터베이스
- 유연한 데이터 모델을 사용하여 데이터를 저장하고 조회
- 데이터는 키-값 쌍, 문서(Document), 그래프(Graph), 열(Column) 등 다양한 형태로 저장
- 데이터의 구조를 미리 정의할 필요가 없으며, 스키마가 동적으로 변경
- 수평적 확장이 용이하며, 대용량 데이터 처리와 빠른 읽기/쓰기 속도를 제공
- 대표적으로 MongoDB, Redis, Cassandra 등이 있습니다.

SQL과 NoSQL 데이터베이스는 각각의 특징과 장단점
SQL 데이터베이스 : 정형화된 데이터와 복잡한 관계를 다루는데 강점을 가지며, ACID 특성을 보장합니다. 
NoSQL 데이터베이스 : 유연하고 확장성이 뛰어나며, 대용량 데이터 처리에 용이
